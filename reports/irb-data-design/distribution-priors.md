# 변수 분포 사전 추정 (distribution-priors.md)

> **상태**: ⛔ IRB 승인 전 — 실제 데이터 미조회. 본 문서의 분포는 **앱 메커니즘·코드에서 역추론한 사전(prior)**이며 *측정값이 아니다*. 종강 후 실분포로 검증·확정한다.
> **목적**: 데이터를 보기 **전에** 변수별 분포 형태와 분석 모형을 사전 확정하여, 결과를 본 뒤 모형을 바꾸는 연구자 자유도(HARKing)를 차단. 책임연구자가 앱 개발자(이해상충)이므로 이 사전 고정은 정직성의 핵심 장치다.
> **신뢰도**: 코드 근거 강함=★★★ / 메커니즘 추론=★★ / 약한 추정=★.

## 2. 피드백(`feedback_count`) 분포 현실 (영과잉 → 신호 둔감)

`feedback_count` 0건 30–50% 추정이 사실이면, "제출자 내 강도" 신호의 **유효 정보 N ≈ 75–105**(148 아님). MDE(1-df, α=.05) 약 f≈0.23은 *명목*이고 영과잉 하에서 실제 탐지력은 더 둔감.
→ 대응(사전 고정·단순): 허들/음이항 *모형* 대신 **2부 기술 보고** — (a) 군집별 *1건+ 제출자 비율*(비율+95%CI), (b) *제출자 중 중앙값(IQR)*, (c) 표본 근거는 검정력 아닌 *정밀도 한계*로만. 모형 적합 없이 기술로 0과잉을 분리해 해석·방어 단순화. `feasibility §5`·`extraction-design §8`과 정합.

## 3. [확정] 게이미피케이션(구 H6) — 활동량 대리·기술 포함 (EXP 결정함수)

코드상 `rabbit_species_owned ≈ min(floor(totalExp/50), 80)`, `totalExp`와 상관 ≈ 0.98 (`rabbitGacha.ts:62-240`, 마일스톤=50EXP 단위). 우리는 `totalExp`·`rank`를 *사용 행동이 아닌 수행·집계 지표이자 종속변수와 동일 도메인 강상관*이라 이미 제외했으므로, 토끼 파생 변수는 같은 교란을 뒷문으로 들이는 일관성 위반.

대체후보 3종을 코드로 재검증(액션 B 완료)한 결과 **전부 EXP 대리**: `equip_slot_utilized`≈0.98 / `gacha_pass_count`≈1.0 / `rabbit_max_level`≈0.70–0.85. 토끼·뽑기·장착·레벨 *전 변수*가 EXP 마일스톤 결정함수라 활동량과 분리 측정 불가.

**확정 결정(C)**: 게이미피케이션은 *독립 가설/차원·동기 측정치가 아님* — 단 **활동량 대리로 기술 프로파일에 포함**한다. ① `rabbit_species_owned`·`gacha_count`: 군집별 기술만, 차이가 보여도 활동량 산물로만 해석(명시). ② EXP-결정성 자체를 본문 4.4에 *방법론적 발견/기여*로 보고. ③ `gacha_choice_ratio`(마일스톤 선택비율, 보유수보다 EXP 의존 약함): **고찰의 후속연구 훅**으로만 — 검증된 동기척도(예: SDT 기반 IMI) 포함 후속연구 제언, *동기 측정·주장 안 함*. 배틀은 기본사용량 기술(`battle_used` 천장 → 기술용, `battle_count` 보고). 본문(4.3·4.4)·data-dictionary·feasibility·메모 동기화 완료.

## 4. ⚠️ [측정 타당도] H7 랭킹 — 신호는 유효하나 가로모드 누락

- **좋은 점**: `ranking_open`은 홈 자동노출이 아니라 **랭킹 상세를 여는 명시적 클릭**에서만 로깅(`RankingSection.tsx:167`). 즉 "능동적 사회비교 조회" 신호로 **타당**(천장효과 우려 해소). 변수 명세의 category는 **`ranking_open`**(`/@overlay/ranking_open`) — variables·data-dictionary·protocol 전 문서 반영 완료.
- **결함**: 가로모드(3패널) 분기(`RankingSection.tsx:165`)는 `RankingBottomSheet`를 열면서 **`logOverlay`를 호출하지 않음** → 가로모드 주 사용 학생의 랭킹 조회가 **체계적으로 미기록**. 기기·사용모드와 상관된 차등 결측.
- **대응(사전 고정)**: ① `ranking_access_intensity`(전체 pageview 정규화) 유지로 노출량 보정, ② 가로모드 사용 비율을 **공변량 또는 한계로 명시**(가로모드 의존 학생일수록 H7 변수 과소측정), ③ H7 결과는 효과크기+CI로 보수적 보고. 누락이 군집과 상관되면 H7 결론 약화 → 한계 명시.

## 5. 사전 확정 사항 요약 (변경 시 사유 기록 의무)

1. **전 변수 동일·단순 표준 절차** = 비모수 Jonckheere–Terpstra 순서대립 추세 + 효과크기(Cliff's δ)·95%CI, **R 패키지**(DescTools·effsize·boot). 정식 ANCOVA·다자유도 옴니버스·교란규칙 없음. 확증 주가설 없는 동등 기술.
2. **피드백(feedback_count)**: 허들 *모형* 대신 **2부 기술**(1건+ 제출비율 / 제출자 중앙값). 영과잉 30–50% 가정, 유효 정보 N 75–105.
3. **게이미피케이션(구 H6)**: 가설/독립차원·동기 아님 — 활동량 대리로 *기술만* 포함. EXP-결정성은 4.4 방법론적 발견. `gacha_choice_ratio`는 고찰 훅(동기 주장 안 함). 배틀은 기본사용량 기술(`battle_count`).
4. **사회비교(랭킹)**: category `ranking_open`(명시적 클릭), 가로모드 미로깅=차등결측 한계, intensity 정규화 유지.
5. **battle_used**: 천장(85–95%)으로 검정 부적합 → 기술용. `battle_count`로 대체 보고.
6. 본 사전은 *추론*이다. 종강 후 실분포가 사전 가정과 달라도 **표준 절차(J-T(R)+효과크기·CI, 0과잉은 2부 기술)는 변경하지 않으며**, 단순 민감도 *기술*만 더해 사유·결과를 함께 보고한다(본문 4.6·9.3 사전 고정과 정합 — 절차 임의 변경 불가).
