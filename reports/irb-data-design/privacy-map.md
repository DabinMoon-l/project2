# 개인정보 위험 매핑 (privacy-map.md)

> **상태**: ⛔ IRB 승인 전 설계 문서. 실제 추출 금지.
> **원칙**: 분석에 필요한 **최소 데이터만**, **비식별 후** 사용. 식별자·자유텍스트·종속변수 중복 필드는 추출 단계에서 차단.

---

## 1. 자유 텍스트 / PII 혼입 가능 필드 식별

연구자가 통제할 수 없는 사용자 입력이 들어가는 필드 = PII·민감정보 혼입 위험.

| 필드 | 위치 | 위험 내용 | 위험도 |
|---|---|---|---|
| `posts.content`, `posts.title` | 게시판 글 | 학생이 본문에 실명·학번·연락처·건강정보 기재 가능 | 상 |
| `comments.content` | 댓글 | 동상 | 상 |
| **콩콩이 비공개 글 본문** (`posts` where `isPrivate=true`) | 나만의 콩콩이 | 개인 학습 고민·사생활. 연구 무관 | **최상** |
| `questionFeedbacks.content` | 문제 피드백 자유서술(선택 필드) | 학생이 자유 서술에 신상·감정 기재 가능 | 상 |
| `pageViews.path` 쿼리스트링 | `?autoStart=`, `?from=` 등 | 일반적으로 라우트뿐이나 쿼리에 식별정보 혼입 이론상 가능 | 중 |
| `users.nickname` | 프로필 | 학생이 실명/학번을 닉네임으로 설정한 사례 다수 | 상 |
| `users.name` (실명) | 등록 정보 | 직접 식별자 | 최상 |
| `users.studentId` (학번) | 등록 정보 | 직접 식별자(조인 키) | 최상 |
| `rabbitHoldings.name` | 토끼 이름(사용자 작명) | 실명/욕설 등 가능, 연구 무관 | 중 |

## 2. 🚫 절대 추출 금지 컬럼 (분석 데이터셋 진입 차단)

추출 스크립트가 **SELECT 자체를 하지 않음**. 카운트가 필요하면 *집계만* 하고 원문은 메모리에 적재하지 않음.

| 컬럼/대상 | 사유 |
|---|---|
| `posts.content` / `posts.title` 원문 | 자유텍스트 PII. 연구는 *건수/유형*만 필요 |
| `comments.content` 원문 | 동상 |
| **콩콩이 비공개 글 전부**(`isPrivate=true`의 모든 필드·본문) | 사생활. 변수에서도 제외(`kongi_private_ratio` 폐기). 카운트조차 추출 안 함 |
| `questionFeedbacks.content` 원문 | 자유서술. `type`(범주)만 사용 |
| `users.name`(실명) | 직접 식별자. 조인 후 즉시 폐기, 데이터셋 미포함 |
| `users.nickname` | 준식별자(실명 혼입 빈번). 분석 불요 → 미추출 |
| `rabbitHoldings.name`(토끼 이름) | 자유텍스트. 종 수 카운트만 |
| **`rank`, `totalExp`** | 사용 행동이 아닌 수행·집계 지표, 종속변수와 동일 도메인 강상관 → 독립변수 부적합. 데이터셋 미포함 |
| **`quizResults.score`/정답률** | **절대 추출 금지** — 수행 지표(행동 아님)·종속변수 동일도메인 상관물. 보조 파일도 미생성. `quizResults`는 시도·재시도 행동량만 |
| `pageViews.path` 쿼리스트링 | `?` 이후 절단 후 라우트만 보존 |

---

## 3. 🔁 해시/카테고리화 후 추출 컬럼

| 컬럼 | 처리 |
|---|---|
| `studentId`(학번) | 조인 키로만 사용 → **즉시 `research_id`(R001~R148)로 치환**. 학번↔R 매핑표는 교수만 보관, 연구자 데이터셋엔 R만 |
| `pageViews.path` | 쿼리스트링 절단 → `category`로 환원. postId/quizId는 **핀 여부·재진입 판정에만 일시 사용 후 폐기**(최종 CSV에 raw ID 미포함) |
| `pageViews.userId` / `posts.authorId` 등 모든 uid | 조인 후 `research_id`로 치환 |
| `questionFeedbacks.type` | 범주값 그대로(자유텍스트 아님, 안전) |
| `class_id` | 그대로(군집 통제용, 준식별이나 4범주라 위험 낮음) |

## 4. 비식별 처리 순서 (불변 규칙)

```
원천 DB
  └─(1) 학번/uid로 조인 (교수 환경 또는 보안 환경에서만)
       └─(2) 학번 → research_id 치환, 매핑표 분리 보관(교수)
            └─(3) 자유텍스트 필드 전면 드롭, 카운트만 산출
                 └─(4) rank/totalExp/정답률 등 중복 필드 드롭
                      └─(5) postId/quizId 등 raw ID 드롭(판정 후)
                           └─ 최종: research_id + 행동 집계값만의 wide CSV
```

- (1)~(2)는 **연구자(앱 개발자) 본인이 직접 수행하지 않음** — 개발자의 로그 접근 권한이 IRB 우려 사항이므로, 비식별 데이터셋만 분석자에게 전달.
- 최종 CSV에는 **재식별 가능한 어떤 키도 없음**.

---

## 5. 잔존 재식별 위험 & 완화

| 위험 | 완화 |
|---|---|
| 소수 군집(예: 특정 분반×하위군집 N<5)에서 행동 패턴으로 개인 추정 | 보고 시 N<5 셀은 통합/마스킹, 개별 행 비공개(집계만 공표) |
| `class_id` + 사용패턴 결합 추정 | 논문/보고에 개인 단위 미공개, 군집 평균만 |
| 시간 패턴(야간/주말)으로 생활 추정 | 비율값만 사용, 원시 타임스탬프 미공개 |

---

## 6. 보관·폐기

- 분석 종료 후 **학번↔research_id 매핑표 파기**(교수, 시점 IRB 계획서 명시).
- 중간 산출물(raw ID 포함 임시 파일)은 비식별 완료 즉시 삭제, 디스크 잔존 금지.
- 최종 비식별 CSV만 분석 기간 보관, IRB 명시 기간 후 폐기.
